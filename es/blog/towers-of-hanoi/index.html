Lo que tenemos que resolver

Existe tres torres y queremos mudar x discos de uno a otro. Sin embargo:

  • No podemos mover más que un disco a la vez.
  • Un disco grande nunca puede cubrir uno más pequeño.

El objectivo es producir un array que representa cada vez que movamos los discos.

Un algoritmo

Mover una pila de un disco o dos es muy facil. Con tres, empieza a complicarse la cosa.

[("A","B"),("A","C"),("B","C"),("A","B"),("C","A"),("C","B"),("A","B")]

Con 4 discos:

[("A","C"),("A","B"),("C","B"),("A","C"),("B","A"),("B","C"),("A","C"),("A","B"),("C","B"),("C","A"),("B","A"),("C","B"),("A","C"),("A","B"),("C","B")]

Existe una paradigma recursiva:

  1. Mover n - 1 discos desde a a c
  2. Mover 1 disco de a a b
  3. Mover n - 1 discos descde c a b

El codigo

Podemos hacer la semántica más comprensivo utilizando sinonimos de tipos:

type Tower = String
type Move = (Tower, Tower)

Y con un signature hacemos una declarácion de como funciona y que resultado tiene el funcion.

hanoi :: Integer -> Tower -> Tower -> Tower -> [Move]

Casos bases

Los principal casos bases:

hanoi 0 _ _ _ =  []
hanoi 1 start end _ =  [(start, end)]

caso recursivo

hanoi n start end temp =
  let nMinusOne = (n - 1)
    in
    hanoi nMinusOne start temp end ++
    hanoi 1 start end temp ++
    hanoi nMinusOne temp end start

Como funciona nuestra recursion:

THis is how we might call it:

hanoi 3 "a" "b" "c"

  • No sale valores, como no hay case base todavia. nMinusOne ahora es 2.

    hanoi 2 “a” “c” “b”

  • Cambiamos el orden de los torres.

  • nMinusOne siendo ahora 1, salen estos valores:

[(“a”,“b”),(“a”,“c”),(“b”,“c”),(“a”,“b”)]

  1. hanoi 1 start temp end produce [(“a”, “b”)]
  2. hanoi 1 start end temp produce [(“a”, “c”)]
  3. hanoi 1 temp end start produce [(“b”, “c”)]
  4. [(“a”, “b”)] - N tengo claro de donde viene.

Otra vez n = 2. Creo que es porque nMinusOne siempre va ser 2. Los argumento ahora son: “c”“b”“a”.

  1. hanoi 1 start temp end produces [(“a”, “b”)]
  2. hanoi 1 start end temp produces [(“c”, “b”)]
  3. hanoi 1 temp end start produces [(“a”, “b”)]
